A. Lambda Function for Event Collection
python# lambda_function.py - Deploy this to AWS Lambda
import json
import boto3
import os
from datetime import datetime

s3 = boto3.client('s3')
S3_BUCKET = os.environ['S3_BUCKET']  # Your S3 bucket
S3_PREFIX = os.environ.get('S3_PREFIX', 'pipeline-events')

def lambda_handler(event, context):
    """Captures CodePipeline events from EventBridge"""
    detail = event.get('detail', {})
    event_time = event.get('time', datetime.utcnow().isoformat())
    
    # Structure the data for Athena
    pipeline_data = {
        'account': event.get('account'),
        'time': event_time,
        'region': event.get('region'),
        'pipeline': detail.get('pipeline'),
        'execution_id': detail.get('execution-id'),
        'start_time': detail.get('start-time', event_time),
        'stage': detail.get('stage', 'Unknown'),
        'action': detail.get('action', 'Unknown'),
        'state': detail.get('state'),
    }
    
    # Write to S3 with date partitioning
    dt = datetime.fromisoformat(event_time.replace('Z', '+00:00'))
    s3_key = f"{S3_PREFIX}/year={dt.year}/month={dt.month:02d}/day={dt.day:02d}/{detail.get('execution-id')}_{dt.strftime('%H%M%S')}.json"
    
    s3.put_object(
        Bucket=S3_BUCKET,
        Key=s3_key,
        Body=json.dumps(pipeline_data)
    )
    
    return {'statusCode': 200}

2. Athena Table Setup
Run this SQL in Athena to create the table:
sql-- Create database
CREATE DATABASE IF NOT EXISTS cicd_analytics;

-- Create external table pointing to S3
CREATE EXTERNAL TABLE IF NOT EXISTS cicd_analytics.pipeline_events (
    account string,
    time string,
    region string,
    pipeline string,
    execution_id string,
    start_time string,
    stage string,
    action string,
    state string
)
PARTITIONED BY (
    year int,
    month int,
    day int
)
STORED AS JSON
LOCATION 's3://your-bucket/pipeline-events/'
;

-- Load existing partitions
MSCK REPAIR TABLE cicd_analytics.pipeline_events;

📊 Data Flow Explanation

CodePipeline executes and emits state change events
EventBridge catches these events based on rule patterns
Lambda transforms events into structured JSON and writes to S3
S3 stores events partitioned by date for efficient querying
Athena provides SQL interface to query S3 data
This App queries Athena and uses Bedrock AI to interpret results
User gets natural language responses about pipeline performance

🔍 What This App Does

Queries: Executes SQL against Athena to retrieve pipeline data
Preprocesses: Cleans and structures data for analysis
Summarizes: Generates statistics about pipeline executions
AI Analysis: Uses AWS Bedrock (Claude) to interpret natural language questions
Chat Interface: Maintains conversation history for contextual discussions

🛠 Troubleshooting
"No data to process"

❌ Lambda not deployed or not triggered
❌ EventBridge rule not configured correctly
❌ S3 bucket permissions issue
✅ Check Lambda CloudWatch logs
✅ Verify EventBridge rule is active
✅ Run test pipeline execution

"Query failed"

❌ Athena table doesn't exist
❌ Wrong database/table name in .env
❌ S3 permissions issue
✅ Verify table exists: SHOW TABLES IN cicd_analytics;
✅ Check IAM permissions for Athena

"Bedrock error"

❌ Model ID incorrect
❌ Region doesn't support Bedrock
❌ No Bedrock access
✅ Enable Bedrock in AWS Console
✅ Check model availability in your region
